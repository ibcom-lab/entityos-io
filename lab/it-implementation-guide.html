Overview

What this achieves

entityOS uses mutual TLS (mTLS) to authenticate users. Instead of passwords, each user carries a short-lived client certificate issued by the entityOS Certificate Authority. This guide covers everything your IT team needs to deploy this across your organisation.

01 — ISSUE

Users self-register

Users visit the registration URL, enter their security code, and a certificate is generated entirely in their browser. The private key never leaves their device.

02 — INSTALL

Certificate is installed

A .p12 file is downloaded and imported into the OS certificate store. The browser detects it automatically — no plugins or extensions required.

03 — TRUST

CA is deployed via MDM

IT pushes the entityOS CA certificate to all managed devices as a trusted root. Users see no warnings and the connection is seamless on every subsequent visit.

Step 1

Issue security codes to users

Each user needs a unique security code before they can register. Security codes are SHA-256 hashed before storage — the plaintext code is given to the user, only its hash is held by the system.

1

Generate a UUID for each user

Use any UUID v4 generator. The format is irrelevant — any string works as a security code. UUIDs are convenient because they are already unique and random.

PowerShell 
[System.Guid]::NewGuid().ToString()
macOS / Linux 
uuidgen | tr '[:upper:]' '[:lower:]'
2

Add the code hash to S3

Use the entityOS cert-setup factory to hash and upload the code in one step. This updates setup/entityos-security-codes.hashes in the registration S3 bucket.

entityos.invoke('util-cert-setup-code-add', {
  code: 'a1b2c3d4-e5f6-7890-abcd-ef1234567890'
});
3

Send the code to the user

Send the plaintext UUID to the user via a secure channel (e.g. encrypted email, IT ticketing system, or in-person handover). Include the registration URL and these instructions.

Security codes are reusable — the same code can register on multiple devices. To revoke a user's access, remove their hash from the S3 hashes file.

Sample user email

Subject: Your entityOS access code

Hi [Name],

To access entityOS-protected systems you need to register a client certificate on each device you use.

Your security code:
a1b2c3d4-e5f6-...

Registration URL:
https://cert.register.entityos.io

Visit the URL, enter your code, and follow the install instructions. Contact IT if you need help.

Step 2

Install the certificate

Users visit cert.register.entityos.io, enter their security code and email, then follow the platform-specific steps below. The .p12 download and install instructions are also shown on the registration page.

1

Visit the registration page

Open cert.register.entityos.io in Chrome, Safari, or Edge. Enter the security code and email address, then click Generate & Register Certificate.

2

Download the .p12 file

The .p12 file downloads automatically. Note the password shown on the page — you will need it in the next step.

3

Import into Keychain Access

Double-click the downloaded .p12 file. Keychain Access will open. Enter the password when prompted and click Add. The certificate appears under My Certificates.

4

Restart the browser

Fully quit and reopen Chrome, Safari, or Edge. On the next visit to an entityOS-protected resource, the browser will present the certificate automatically.

Certificate not trusted? The registration page includes a one-click download of the entityOS CA certificate. Alternatively, IT can push the CA cert to all Macs via MDM — see Step 3.

1

Visit the registration page

Open cert.register.entityos.io in Chrome or Edge. Enter the security code and email, then click Generate & Register Certificate.

2

Download the .p12 file

The .p12 file downloads automatically. Note the password shown on the page.

3

Run the Certificate Import Wizard

Double-click the .p12 file. The Certificate Import Wizard opens.

  • Select Current User as the store location → Next
  • Confirm the file path → Next
  • Enter the .p12 password → Next
  • Select Automatically select the certificate store → Next
  • Click Finish
4

Verify in Certificate Manager

Open certmgr.msc → Personal → Certificates. Confirm the entityOS certificate appears.

# Open from Run dialog (Win+R):
certmgr.msc
5

Restart the browser

Fully quit and reopen Chrome or Edge. The certificate will be presented automatically on entityOS-protected resources.

Firefox on Windows maintains its own certificate store. See the registration page for Firefox-specific instructions if your organisation uses Firefox.

1

Open the registration page in Safari

Navigate to cert.register.entityos.io in Safari on the iPhone or iPad. Enter the security code and email, then tap Generate & Register Certificate.

2

Download the .p12

Tap the download link on the page. iOS will prompt to save the file. Note the password shown on screen.

Alternatively, email or AirDrop the .p12 from a Mac or Windows machine if the browser auto-download does not trigger correctly.

3

Install the profile

Tap the .p12 in Files or in the email attachment. iOS will say "Profile Downloaded". Then:

  • Go to Settings → General → VPN & Device Management
  • Tap the downloaded profile under Downloaded Profile
  • Tap Install → enter device passcode
  • Enter the .p12 certificate password when prompted
  • Tap Done
4

Safari picks it up automatically

No browser restart needed on iOS. Safari will present the certificate on the next visit to an entityOS-protected resource.

1

Open the registration page in Chrome

Navigate to cert.register.entityos.io in Chrome on Android. Enter the security code and email, then tap Generate & Register Certificate.

2

Download the .p12

The .p12 downloads to the Downloads folder. Note the password on screen.

3

Install the certificate

Path varies slightly by manufacturer, but generally:

  • Settings → Security → Encryption & credentials
  • Tap Install a certificate → VPN & app certificate
  • Browse to and select the .p12 file from Downloads
  • Enter the .p12 password when prompted
  • Give it a name (e.g. entityOS) and tap OK

On Samsung devices the path is Settings → Biometrics and security → Other security settings → Install from device storage.

4

Chrome picks it up automatically

Chrome on Android uses the system certificate store. Restart Chrome if the certificate is not presented immediately.

Step 3

Deploy CA trust via MDM

Without the entityOS CA in the device trust store, users will see a "certificate not trusted" warning. Distributing the CA cert via MDM makes trust silent and automatic — zero user interaction required.

Apple / Microsoft

Microsoft Intune

Devices → Configuration profiles → Create → macOS or Windows → Templates → Trusted certificate. Upload the CA PEM. Assign to device group → Deploy.

Apple-first

Jamf Pro

Configuration Profiles → New → Certificate payload. Upload the CA PEM. Scope to your device group → Save → push. Works for macOS and iOS.

Apple SMB

Mosyle

Management → Profiles → Add Profile → Certificate. Upload the CA PEM → assign to devices. Lightest-touch MDM for Apple-only environments.

Manual alternative (no MDM)

Trust the CA manually per device

Download the entityOS CA certificate from the registration page (or obtain it from your IT team), then follow the platform steps below.

▸

macOS

Double-click the entityOS-CA.pem file → Keychain Access opens → expand Trust → set When using this certificate to Always Trust → enter Mac password to confirm.

▸

Windows

Double-click entityOS-CA.pem → Certificate Import Wizard → place in Trusted Root Certification Authorities → Finish. Or run:

certutil -addstore "Root" entityOS-CA.pem
▸

iOS / iPadOS

Download entityOS-CA.pem → Settings → General → VPN & Device Management → install the profile → then go to Settings → General → About → Certificate Trust Settings and enable full trust for the entityOS CA.

Ongoing

Certificate renewal

Certificates are valid for 5 days. Users need to re-register before their certificate expires, or access to entityOS-protected resources will be denied.

Recommended: notify users before expiry

Send users a renewal reminder 1–2 days before their certificate expires. The renewal process is identical to initial registration — visit the URL, enter the code, download and install the new .p12.

Each renewal generates a new unique CN (entityOS-{hex}) so Keychain accumulates entries over time. Old expired certs can be safely deleted from the keychain at any time.

Revoking access

To revoke a user's ability to re-register, remove their code hash from the S3 hashes file. Their current certificate will remain valid until it expires (max 5 days), after which it cannot be renewed.

entityos.invoke('util-cert-setup-code-remove', {
  code: 'a1b2c3d4-e5f6-...'
});
Deployment checklist

Before go-live

IT / Infrastructure

  • entityOS CA certificate deployed to all managed devices via MDM
  • S3 setup bucket contains setup/entityos-security-codes.hashes
  • Lambda IAM roles have s3:GetObject and s3:ListBucket on setup/*
  • Security codes issued to all users and stored securely in IT records
  • Renewal reminder process documented and scheduled
  • Revocation process tested with a test user

End user

  • Registration URL and security code received
  • Certificate generated and .p12 downloaded
  • .p12 imported into OS certificate store with correct password
  • entityOS CA certificate trusted (via MDM push or manual install)
  • Browser restarted after import
  • Confirmed access to an entityOS-protected resource
